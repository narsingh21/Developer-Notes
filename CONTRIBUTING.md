# Contributing to Developer Learning Notes

Thank you for your interest in contributing! This repository is meant to be a community-driven resource for developers.

## How to Contribute

### Adding New Content

1. **Fork the repository**
2. **Create a new branch** for your feature or fix:

   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **Add your content** in the appropriate folder:
   - `React.Js/` - React-related notes
   - `Node.Js/` - Node.js backend content
   - `Java Script/` - JavaScript concepts
   - `Data structure Algorithms/` - DSA notes
   - `Frontend System Design/` - System design documents
   - `GraphQl/` - GraphQL content
   - `microservices/` - Microservices architecture
   - `Fronted-Questions/` - Interview questions

4. **Follow these formatting guidelines:**
   - Use `.md` extension for markdown files
   - Use clear, descriptive filenames
   - Add proper headings and sections
   - Include code examples where appropriate

5. **Commit your changes** with a descriptive message:

   ```bash
   git commit -m "Add: React hooks best practices"
   ```

6. **Push to your fork**:

   ```bash
   git push origin feature/your-feature-name
   ```

7. **Submit a Pull Request**

## Content Guidelines

- Keep notes clear and concise
- Include practical code examples
- Use proper markdown formatting
- Add table of contents for longer documents
- Include references when applicable

## Report Issues

Found a mistake or have a suggestion? Open an issue with:

- Clear description of the problem
- Suggested fix (if any)
- Relevant file paths

## Questions?

Feel free to open an issue for any questions about contributing.
